\## 목표

\## AI 대화 기록



\*\*Q1: 프레임 수를 게임을 실행하며 알고 싶어. 프레임 수를 화면에 띄우는 코드를 알려줘



\-AI 답변: font = while위 pygame.font.SysFont(None, 30)

&#x20;                     while 안 fps = clock.get\_fps()

&#x20;                                 fps\_text = font.render(f"FPS: {int(fps)}", True, (0, 0, 0))

&#x20;                                 screen.blit(fps\_text, (10, 10)) 코드 사용

\-시행착오: fps문구가 뜨지 않음

\-최종 해결: 실수로 while 안에 넣어야하는 코드를 넣지 않음. 코드를 다시 넣고 실행해 성공



\*\*Q2: 방향키로 동그라미를 움직이게 하는 코드를 짜줘



\-AI 답변: keys = pygame.key.get\_pressed() 사용

\-시행착오: 움직임 자체는 성공이나 대각선 이동시 벡터값이 정규화 되지 않아 속력이 빨라짐

\-최종 해결: length = math.sqrt(dx\*\*2 + dy\*\*2)

&#x20;              dx = dx / length

&#x20;              dy = dy / length 을 사용해 해결



\*\*Q3: shift키를 누르면 속도가 2배가 되도록 코드를 짜줘



\-AI답변: if keys\[pygame.K\_LSHIFT] or keys\[pygame.K\_RSHIFT]:

&#x20;   		current\_speed = speed \* 2을 사용

\-시행착오: 코드의 어느부분에 넣어야하는지 모름

\-최종 해결: 플레이어 캐릭터를 움직이게 하는 코드 밑에 넣어 해결



\#배운점

이번 실습을 통해 AI에게 질문하고 문제를 해결하는 과정을 거치며 처음보는 생소한 개념도 빠르게 이해할 수 있었다. 

기억에 남는 코드로 플레이어 캐릭터가 대각선으로 움직일 때 속력이 빨라지는 것을 고치는 코드가 가장 기억에 남았다. 처음 코드만 봤을 때는 dx, dy가 있어 익숙한 느낌이 들면서도 length, sqrt라는 생소한 단어들이 있어 이해가 가지 않았다. 하지만 AI에게 질문을 해가며 단어들이 의미하는 바를 알고 코드를 이해하니 코딩과 한 걸음 더 가까워진 느낌이 들었다. 특히 dx = dx / length 부분이 이해가 가지 않았는데 length를 이해하고 나니 퍼즐이 맞춰지듯 이해할 수 있었다.
