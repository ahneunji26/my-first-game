\## 관찰 내용



\### 원형 vs AABB

동그란 모양의 스프라이트를 사용했을 때에 원형의 경우 빈 공간이 거의 없어 충돌 판정이 거의 정확했다. AABB의 경우 원형보다 모서리 쪽에 빈 공간이 더 많아 부딪히지 않았는데도 충돌이 되었다.



\### AABB vs OBB

AABB의 경우 회전하는 스프라이트의 최대 길이에 맞춰 박스의 크기가 진동하듯 커졌다 작아졌다를 반복했다. OBB의 경우 회전하는 스프라이트에 맞춰 박스가 같이 회전했다.



\## AI와의 대화에서 배운 것

\-sprites를 파이게임에 넣으려면 sprites.py라는 파일을 따로 만들어 넣어야한다는 것을 알았다.

\-흥미로웠던 점으로 대각선 속력이 빨라져 벡터 정규화를 하는데 나타내려는 속력이 5라고 했을 때 벡터 정규화를 하고 대각선의 속력이 5보다 더 작아졌던 것이 있다.  AI답변으로

int(dx \* speed)를 매 프레임 바로 적용하면 소수점이 버려짐

float 좌표를 따로 저장하면 정상 속도가 나옵니다

라는 답변을 받았다.



\## 내 게임에 적용한다면

나는 AABB의 방식을 주로 사용할 것 같다. 왜냐하면 내가 만들고 싶은 게임은 2d 플랫포머 게임으로, 캐릭터가 바닥이나 벽, 장애물에 닿는가가 중요하므로 OBB같이 구현이 세세한 것 보다는 AABB가 더 좋을 것이라 생각했다. 하지만 회전을 하는 적 캐릭터라던가 원형 모양의 코인 오브젝트에서는 AABB가 적합하지 않은 경우도 있으니 캐릭터나 오브젝트의 특성을 잘 고려해서 사용해야 할 것 같다.
